# Asset Header Generator

## Description

This project provides two Python scripts for embedding binary files into C++ header files, and reversing that process:

- **GenerateAssetHeader.py** converts a binary file (image, audio, or any other binary asset) into a C++ header file. The binary data is written out as a byte array along with a small `EmbeddedAsset` struct that packages a version string, a pointer to the data, and the data size. This lets you compile binary assets directly into an executable instead of shipping them as separate files.
- **GenerateBinaryFile.py** reverses the process, reading a previously generated header file and reconstructing the original binary file from the byte array it contains.

Each generated header guards against duplicate struct definitions using an `#ifndef` include guard, and marks the array and struct instance as `inline` so the header can safely be included in multiple translation units.

## Requirements

- Python 3

No external dependencies are required; both scripts use only the Python standard library.

## Usage

### Converting a binary file to a header

```
python GenerateAssetHeader.py <input_binary_file> [output_header_file]
```

- `input_binary_file`: path to the binary file to embed (required).
- `output_header_file`: path for the generated header (optional). If omitted, the output file uses the same name as the input file with a `.h` extension.

Example:

```
python GenerateAssetHeader.py icon.png icon.h
```

This produces `icon.h`, containing:

- An `EmbeddedAsset` struct definition (guarded against redefinition).
- A raw byte array named `icon_png_raw_data`.
- An `EmbeddedAsset` instance named `icon_png` that packages the version string, the data pointer, and the data size.

### Converting a header back into a binary file

```
python GenerateBinaryFile.py <input_header_file> [output_binary_file]
```

- `input_header_file`: path to a header file previously generated by `GenerateAssetHeader.py` (required).
- `output_binary_file`: path for the restored binary file (optional). If omitted, the output file uses the same name as the input file with a `.bin` extension.

Example:

```
python GenerateBinaryFile.py icon.h icon_restored.bin
```

This reads the `_raw_data` array from the header file and writes the original bytes back out to disk.

## Notes

- The generated variable name is derived from the input file name, with any non-alphanumeric characters replaced by underscores.
- `GenerateBinaryFile.py` looks specifically for an array named with the `_raw_data` suffix, so it expects headers produced by `GenerateAssetHeader.py` or files following the same format.
- Both scripts print a success message with the resulting file path and size, or an error message if the input file cannot be found or parsed.
